# 路由和侧边栏

路由和侧边栏是组织起一个后台应用的关键骨架。 本项目侧边栏和路由是绑定在一起的，所以只有在@/router/index.js下面配置对应的路由，侧边栏就能动态的生成，不过配置路由需要遵循一定约定的规则

# 配置项

路由配置项

//当设置 true 的时候该路由不会再侧边栏出现 如401，login等页面/1
hidden: true // (默认 false)

//当你一个路由下面的 children 声明的路由大于1个时，自动会变成嵌套的模式--如组件页面
//只有一个时，会将那个子路由当做根路由显示在侧边栏--如引导页面

//若你想不管路由下面的 children 声明的个数都显示你的根路由
//你可以设置 alwaysShow: true，这样它就会忽略之前定义的规则，一直显示根路由
alwaysShow: true

name: 'router-name' //设定路由的名字，一定要填写不然使用<keep-alive>时会出现各种问题
meta: {
  title: 'title' // 设置页面的标题
  icon: 'svg-name' //设置该路由的图标
}
rootRouter: [ // 插到跟路径下的，属于 root 的子路由
  {
    path: 'addUser',
    name: 'addUser',
    meta: {
      title: '添加新用户'
    },
    component: () => import('@/views/permission/addUser')
  }
],

示例

{
    path: '/permission',
    name: 'permission',
    component: Root,
    children: [
      {
        path: 'userManger',
        name: 'userManger',
        meta: {
          title: '用户管理'
        },
        component: () => import('@/views/permission/userManger'),
        children: [
          {
            path: 'userManger-1',
            name: 'userManger-1',
            alwaysShow: true,
            component: () => import('@/views/permission/userManger-1'),
            rootRouter: [
              {
                path: 'addUser',
                name: 'addUser',
                meta: {
                  title: '添加新用户'
                },
                component: () => import('@/views/permission/addUser')
              }
            ],
            children: [
              {
                hidden: true,
                path: 'userManger-1-1',
                name: 'userManger-1-1',
                component: () => import('@/views/permission/userManger-1-1')
              },
              {
                hidden: true,
                path: 'userManger-1-2',
                name: 'userManger-1-2'
              }
            ]
          },
          {
            path: 'userManger-2',
            name: 'userManger-2',
            component: () => import('@/views/permission/userManger-2')
          }
      ]
 }

# 路由

这里的路由分为三种，constantRouterMap、asyncRouterMap和rootRouters

constantRouterMap： 代表那些不需要动态判断权限的路由，如登录页，错误页面

asyncRouterMap： 代表那些需要动态判断权限并通过addRouters动态添加的页面

rootRouters： 代表那些路由里面rootRouter选项，添加在这个里面，页面展示到为Main

# 侧边栏

本项目侧边栏主要基于ant-design-vue的a-menu。

侧边栏是通过读取路由并且结合权限判断而动态生成的，而且还需要支持路由无线嵌套，所以这里还使用了递归组件。

TIP

代码地址 @/views/layout/components/Sidebar

一般侧边栏有两种形式：submenu和a-menu-item，一个是嵌套子菜单，另外一个则是一个链接，如下图

在Sidebar中已经帮你做了判断，当你一个路由下面的children声明的路由大于1个时，并且没有hidden属性，会自动变成嵌套的模式。如果子路由正好等于一个就会默认将子路由作为根路由显示在侧边栏中。
