# 配置文件

配置文件都是在@/config文件夹下面，项目初始的时候只有三个文件

menu: 侧边栏配置文件，需要显示的都要配置在这边，格式为对象

// 对象的属性为路由列表的name，属性值为其对应的中文名字
export default {
  home: '首页',
  permission: '权限管理',
  userManger: '用户管理',
  roleManger: '角色管理'
}

WARNING

侧边栏不需要显示的不能配置在这里，否则页面刷新会存在bug(如果Main下面还有路由，侧边栏找不到当前选中的)

realRoute: 路径配置的地址，也就是当前页面路径地址，比如当前的路径是/permission/roleManger，就需要将当前的 roleManger配置到这个文件中

export default {
  detail: '详情',
  roleManger: '角色管理'
}

WARNING

不是path中最后一项的不能配置在这里，否则存在bug(会造成面包屑不是真实的路径可以点击)

blacklist: 不需要配置的缓存的列表，如详情页

export default ['detail']
