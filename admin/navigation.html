# 标签栏导航

由于使用标签栏导航，就需要缓存数据，这里运用keep-alive和router-view

代码: @/views/Root.vue

<keep-alive :include="routeLabelList">
     <router-view></router-view>
</keep-alive>

其中routeLabelList这个列表是动态，会监听路由的变化做出相应的改变，判断当前页面是否需要重新加载或者已被缓存， 这个数据只需要从store中获取就可以，具体的代码在@/store/modules/navigation

WARNING

当前路由中Main是一个路由，底下是没有二级路由的，如果有路由，也是需要配置相应的keep-alive,不然是没法缓存信息的

# 注意事项

由于目前keep-alive和router-view是强耦合的，而且查看文档和源码不难发现keep-alive的include默认是优先匹配组件的name， 所以在编写路由router和路由对应的 view component 的时候一定要确保 两者的name完全一致的。(切记 name 命名时候尽量保 证唯一性 切记不要和某些组件的命名重复了，不然会递归引用最后内存溢出等问题)

DEMO:

// router 路由声明
{
    path: 'roleManger',
    name: 'roleManger',
    meta: {
      title: '角色管理'
    },
    component: () => import('@/views/permission/roleManger')
}

// 路由对应的view
export default {
  name: 'roleManger'
}

# 缓存不适合的场景

目前缓存的方案对于某些业务是不适合的，比如详情，所以这种情况是不需要配置缓存的，在@/config/blacklist.js中，这个文件将过滤哪些不需要配置缓存的页面
